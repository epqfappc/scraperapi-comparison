## 被封 IP 和验证码搞崩溃？你不是一个人

做数据采集的人都经历过这些场景：

1. **凌晨跑批量任务，早上起来发现 80% 请求返回 403**——目标站半夜更新了反爬策略，你的静态代理池一夜之间全废了。
2. **Google SERP 抓取隔三差五触发 reCAPTCHA**——手动解一次验证码要 5-10 秒，一天几万条查询根本不可能人工介入。
3. **电商价格监控需要渲染 JavaScript**——普通 HTTP 代理拿到的是空壳 HTML，有效数据全在动态加载里。
4. **预算有限但目标站分布在多个国家**——买住宅代理按 GB 计费，跑一天流量费比数据本身还贵。

这些痛点的共同根源是：你需要的不只是"一个代理 IP"，而是一套帮你处理轮换、重试、渲染、地理定位的完整基础设施。Proxy API 这个品类就是为了解决这件事而存在的。

## 地理定位和 Session 保持：两个容易踩坑的功能

### 地理定位

ScraperAPI 支持 50+ 国家的IP 定位，加 `&country_code=xx` 就行。但要注意：指定国家的请求成功率会比不指定时低一些，因为可用 IP 池变小了。我测过指定日本节点抓乐天，成功率大概 92%，比不指定时的 96% 低了几个点。如果你的目标站对地理位置不敏感，建议不加这个参数，让 ScraperAPI 自己选最优 IP。

### Session 保持

有些场景你需要多个请求用同一个 IP（比如登录态保持、翻页），ScraperAPI 的 session 功能可以在 5 分钟内保持同一 IP。但超过 5 分钟 session 就会失效，这个限制在做长流程爬取时要注意规划请求节奏。

## 和其他 Proxy API 方案的核心差异

市面上做类似事情的还有 Bright Data、Oxylabs、ZenRows、Smartproxy 等。我不打算做一个面俱到的横评（那得写一本书），只说几个我实际对比过的关键差异：

- **计费模式**：ScraperAPI 按请求次数，Bright Data/Oxylabs 主要按流量（GB）。如果你抓的页面体积大（图片多、DOM 复杂），按次数计费更可控；如果你抓的是轻量 API 接口，按流量可能更便宜。
- **上手门槛**：ScraperAPI 的 API 设计非常简单，一个 URL 参数搞定一切。Bright Data 功能更强但配置也更复杂，适合有专职运维的团队。
- **价格起点**：ScraperAPI 的 Hobby 套餐年付 $29/月拿 10 万次请求，对个人开发者和小团队很友好。Bright Data 的起步价通常更高。
- **适用规模**：超过 500 万次/月的大规模采集，Enterprise 级方案（Bright Data、Oxylabs）的基础设施更成熟；300 万以内 ScraperAPI 完全够用。

## 我踩过的坑和解决办法

**坑 1：JS 渲染请求消耗 10 倍额度**

刚开始用的时候没仔细看文档，给所有请求都加了 `render=true`，结果一天就把半个月的额度烧完了。后来我做了分流：先用普通请求试一次，如果返回的 HTML 里目标数据为空，再用渲染请求重试。这样渲染请求的比例从 100% 降到了 15% 左右。

**坑 2：并发数打满时的排队延迟**

Startup 套餐 50 并发，我有一次同时启动了 3 个采集任务，总并发瞬间超了。超出的请求不会报错，而是排队等待，导致响应时间从 4 秒飙到 30秒+。解决办法是在代码层做并发控制，用信号量把总并发限制在套餐上限的 80%。

**坑 3：某些目标站需要自定义 Header**

ScraperAPI 默认会伪装常见浏览器的 User-Agent 和 Header，但有些站会检查特定的 Cookie 或 Referer。这时候需要用它的 `keep_headers=true` 参数配合自定义 Header 传入。文档里这部分写得不太显眼，我是翻了 GitHub 上的 issue 才找到的。

## 常见问题 FAQ

**Q1：ScraperAPI 免费套餐够用吗？**

5000 次/月只够验证可行性和跑 demo。如果你有实际生产需求，至少需要 Hobby 套餐。好处是免费层不绑信用卡，不会被意外扣费。

**Q2：请求失败会扣额度吗？**

不会。只有返回 200 的成功请求才计入额度消耗。超时、被封、服务端错误都不扣。这一点比很多竞品厚道。

**Q3：支持哪些编程语言？**

本质上是 HTTP API，任何能发 HTTP 请求的语言都能用。官方提供 Python、Node.js、Ruby、PHP 的 SDK，Scrapy 有专用中间件。

**Q4：年付套餐中途能退款吗？**

ScraperAPI 提供 7 天退款窗口。我没亲自试过年付退款，但月付套餐随时可以取消，下个计费周期不再扣费。建议先月付跑一个月确认满足需求，再切年付锁定折扣。

**Q5：抓取速度能有多快？**

取决于目标站响应速度和你是否开启 JS 渲染。普通请求中位延迟 2-5 秒，JS 渲染 8-15 秒。并发数是硬上限，Startup 套餐 50 并发意味着理论峰值吞吐约 10-25 次/秒（普通请求）。

**Q6：能抓需要登录的页面吗？**

可以，通过 session 保持 + 自定义 Cookie Header 实现。但 ScraperAPI 不会帮你处理登录流程本身，你需要自己拿到登录后的 Cookie 再传进去。
